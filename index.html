What I Learned

Real lessons from three weeks of building.

01
Treat it like a person, not a tool
The more context I gave Kip ‚Äî about my goals, my constraints, my ADHD, my style ‚Äî the better everything got. The relationship matters. Write the soul file.
02
Security gates are not optional
After a prompt injection attempt, we built a mandatory codeword ("GO verstappen") for all outbound actions. You're giving the AI access to your email. Treat it like giving someone your keys.
03
File systems are AI memory
Kip doesn't have persistent memory by default. But it reads and writes files. So we built a memory system: MEMORY.md, daily logs, heartbeat state. That's the brain.
04
AI writing has tells ‚Äî fix them
We built a whole guide (AI-WRITING-TELLS.md) of patterns to avoid: em-dashes, "delve," colon-into-lists, balanced framing. If you use AI to write, you have to teach it your voice.
05
Automation compounds faster than you think
The first cron job (morning briefing) took an hour to set up. By week two I had fifteen running. Each one built on the last. The system got smarter the more I used it.
06
ADHD and AI agents are a power combo
I built Kip around how my brain actually works: max 3 priorities, broken-down tasks, external accountability. It doesn't judge me for forgetting. It just reminds me ‚Äî gently, in context.
